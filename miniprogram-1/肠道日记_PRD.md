# 肠道日记 · 排便健康记录小程序

***

## 一、产品定位与目标

### 产品定位

「肠道日记」是一款面向年轻人的肠道健康自我管理微信小程序，帮助用户科学记录每日排便状况，通过数据可视化和健康分析，提升用户对肠道健康的关注意识，培养规律的排便习惯。

### 产品目标

- **核心目标**：提供便捷、无负担的排便记录工具，让用户养成每日打卡习惯
- **健康目标**：通过趋势分析帮助用户识别异常排便规律，辅助健康自查
- **体验目标**：以轻松、去污名化的设计语言降低用户的使用心理门槛
- **数据目标**：上线 3 个月内，日活用户记录完成率 ≥ 60%，次月留存率 ≥ 40%

### 产品价值主张

> 「每天一记，读懂你的肚子。」

## 二、目标用户

### 核心用户画像

| 维度 | 描述                      |
| -- | ----------------------- |
| 年龄 | 18–35 岁                 |
| 场景 | 久坐办公、饮食不规律、有轻度便秘/腹泻困扰   |
| 动机 | 关注健康、有自我量化习惯、对身体指标有记录意愿 |
| 设备 | 微信用户，手机端为主              |

### 用户分层

**A 类 · 健康记录型**

- 对自身健康数据有主动管理意识
- 喜欢使用健康类 App 进行打卡记录
- 期望看到趋势图表和分析报告

**B 类 · 问题驱动型**

- 有便秘、腹泻或肠易激综合征（IBS）等轻度困扰
- 需要在就医时提供近期排便记录给医生参考
- 功能需求聚焦于记录完整性和导出能力

**C 类 · 轻量尝鲜型**

- 被社交分享或好奇心驱动下载
- 使用频率较低，需要通知提醒激活

### 用户痛点

- 排便异常难以量化描述，就医时无法准确告知医生
- 现有健康 App 无专门的肠道记录模块
- 记录流程繁琐，长期坚持难度高

## 五、页面说明（逐页说明）

### 5.1 首页

**核心目的**：让用户快速了解今日状态并完成记录。

**页面布局**：

- **顶部**：问候语（「早上好，今天肠胃怎么样？」）+ 日期
- **打卡状态卡片**：今日是否已记录，若已记录显示最近一条记录摘要（时间 + 形态图标）；若未记录显示「还没记录，现在去记」
- **连续打卡 Banner**：火焰图标 + 「已连续记录 X 天」，鼓励坚持
- **本周小结**：小型柱状图，展示本周每日次数
- **悬浮按钮（FAB）**：右下角圆形「＋」按钮，点击进入记录页

**交互说明**：

- 今日已有记录时，点击记录卡片跳转历史详情
- FAB 按钮常驻显示，任何滚动位置均可触达

***

### 5.2 记录页（新增/编辑）

**核心目的**：3 步内完成一次排便记录，降低填写负担。

**页面布局（步骤式表单）**：

**Step 1 · 基本信息**

- 时间：默认当前时间，点击弹出时间选择器
- 时长：横向滑动条，0–60 分钟，默认跳过

**Step 2 · 大便状态**

- 布里斯托分类图：7 种类型图文展示（卡片式，横向滚动或 Grid 展示）
  - Type 1：坚硬颗粒状（便秘）
  - Type 2：香肠状有硬块
  - Type 3：香肠状有裂纹（理想偏干）
  - Type 4：香肠状光滑（理想）
  - Type 5：柔软小块（偏软）
  - Type 6：糊状（轻度腹泻）
  - Type 7：水状（腹泻）
- 颜色：6 色圆形色块选择

**Step 3 · 感受与备注**

- 感受标签：多选胶囊标签
- 备注：可选文本域

**底部**：「保存记录」主按钮

***

### 5.3 统计页

**核心目的**：可视化呈现肠道健康趋势，帮助用户发现规律。

**页面布局**：

- **切换栏**：日 / 周 / 月（Segment Control）
- **频次图表**：柱状图，X 轴为日期，Y 轴为次数，均值参考线标注（正常范围 1–3 次/天）
- **形态分布**：饼图或色块分布，展示近期最常见的布里斯托类型
- **规律性卡片**：
  - 平均排便时间：「通常在 上午 8–9 点」
  - 平均间隔：「大约每 X 小时一次」
  - 本月总计：X 次
- **健康评估**：彩色标签（绿色·规律 / 黄色·偏少 / 橙色·偏多 / 红色·需关注）+ 简短文字建议

***

### 5.4 历史记录页

**核心目的**：查询、管理所有历史记录。

**页面布局**：

- **顶部**：日历视图，有记录的日期显示绿色圆点，多条记录显示数字角标
- **列表区域**：点击日历日期，下方滚动到对应日期分组；列表项展示：时间 + 形态图标 + 颜色色块 + 感受标签
- **右滑操作**：滑动列表项显示「编辑」「删除」操作

***

### 5.5 我的页

**核心目的**：设置与账号管理。

**页面布局**：

- **用户信息区**：微信头像 + 昵称 + 注册天数
- **设置列表**：
  - 提醒设置（开关 + 时间选择）
  - 数据导出（生成 PDF）
  - 隐私设置
  - 清除数据
  - 关于本小程序
  - 用户反馈

## 六、UI 配色与视觉规范

### 6.1 设计风格

轻量、温暖、去医疗化。避免冷色调和严肃的医疗风格，使用柔和的自然色系，营造「健康生活方式」氛围，降低用户的心理负担。

### 6.2 主色调

| 名称      | 色值        | 用途              |
| ------- | --------- | --------------- |
| 主色 · 暖绿 | `#4CAF7D` | 主按钮、强调元素、健康正向状态 |
| 辅色 · 暖橙 | `#FF8A50` | 提醒、需关注状态、高亮数据   |
| 背景 · 米白 | `#F7F5F2` | 页面背景，柔和不刺眼      |
| 卡片背景    | `#FFFFFF` | 卡片、弹窗背景         |
| 主文字     | `#2D2D2D` | 主要文字            |
| 次文字     | `#8C8C8C` | 辅助说明、时间戳        |
| 分割线     | `#EDEDED` | 列表分割            |

### 6.3 状态色

| 状态     | 色值           |
| ------ | ------------ |
| 健康·规律  | `#4CAF7D`（绿） |
| 偏少/偏干  | `#FFC043`（黄） |
| 偏多/偏稀  | `#FF8A50`（橙） |
| 需关注/异常 | `#F44336`（红） |

### 6.4 字体规范

| 层级   | 字号   | 字重      | 用途    |
| ---- | ---- | ------- | ----- |
| 大标题  | 20px | Bold    | 页面标题  |
| 卡片标题 | 16px | Medium  | 卡片主信息 |
| 正文   | 14px | Regular | 描述文字  |
| 辅助说明 | 12px | Regular | 时间、标签 |

### 6.5 图标与插图

- 布里斯托分类使用扁平化、卡通化插图，弱化直接的写实表现
- 图标统一使用线性风格（2px 描边），圆角处理
- 小程序整体圆角规范：卡片 12px，按钮 24px（胶囊型），标签 16px

## 七、交互与状态设计

### 7.1 记录流程交互

- 点击 FAB 按钮：底部 Sheet 上滑，进入记录表单（非跳页，减少跳出感）
- 布里斯托选择：支持单选高亮，选中项放大并显示绿色边框
- 保存时：轻震动反馈 + Toast 提示「记录成功 ✓」+ FAB 按钮变色为绿色（今日已记录状态）
- 必填项为「时间」和「形态」，其余可跳过

### 7.2 空状态设计

| 场景     | 展示内容                     |
| ------ | ------------------------ |
| 今日无记录  | 插图 + 「今天还没有记录，肠道等你关注 🌿」 |
| 历史无数据  | 插图 + 「开始第一条记录，了解自己的肠道节律」 |
| 统计数据不足 | 「数据积累中，记录满 7 天后可查看趋势」    |

### 7.3 异常状态提示

- 若连续 3 天记录均为 Type 1/2（严重便秘）或 Type 6/7（腹泻），统计页出现橙色提示卡片：「近期肠道状态需关注，建议调整饮食或咨询医生」
- 提示卡片提供「了解改善建议」入口（科普文章链接）和「忽略」选项

### 7.4 加载与反馈

- 图表加载：使用骨架屏（Skeleton），避免白屏
- 数据保存：乐观更新，先显示成功，失败时回滚并提示
- 删除操作：需二次确认弹窗，防止误删
- 网络断开：本地缓存优先，恢复网络后自动同步

### 7.5 权限申请

- 通知权限：首次使用时引导申请，说明用于每日记录提醒
- 微信登录：首次使用统计/导出功能时引导授权，非强制
- 拒绝授权后仍可使用本地记录功能，不降级核心体验
